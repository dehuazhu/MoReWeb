Requirements

  • Python 2.6 (2.7 recommended)
  • optional python modules: MySQLdb
  • ROOT 5.34.19+ or ROOT 6
  • recent Web browser (min Firefox 4, Chrome 18, Safari 3, Opera 9, IE 10)

How to install

  • Clone from git: https://github.com/psi46/MoReWeb.git
  • copy Paths.cfg.default to Paths.cfg and adjust paths

How to run

  • run './Controller.py' in Analyse subdirectory, to analyze all modules, which have not been analyzed with the latest version
  • run './Controller.py -p' to create production overview page
  • run './Controller.py -new' to analyze only modules, which have not been analyzed yet. (don't re-analyze)
  • run './Controller.py -f' to analyze only modules, even if already analyzed with latest version
  • run './Controller.py -FQ /path/to/M1234_...._123456789/' to analyze only a specific Qualification
  • run './Controller.py -m M1234,M5678,M7890' to analyze all Qualifications for listed modules

FAQ and use cases

How can modules uploaded to database if test was not possible at all?

  • See the examples in examples/ subfolder.
  • If a module was not qualified because the leakage current at startup was too high, a LeakageCurrentPON test (see LeakageCurrentPON_IfLeakageCurrentTooHigh.tar example) can be uploaded instead of the Fullqualification.

How is it decided if a folder placed in GlobalDataDirectory is re-analyzed when running MoReWeb?

  • MoReWeb places a checksum of the current code in the FinalResults folder. If the code is changed, either by git pull or manuel changes, all folders will be re-analyzed. To force re-analysis, use the --force (or -f) parameter.

How can one re-fit?

  • MoReWeb does not re-calculate the fits for S-curves and PH, use -r parameter to re-fit in something has changed in the fit procedure.

How can the order of the qualification list be changed?

  • Add a UserConfiguration.cfg in Configuration folder, add section [SystemConfiguration] with entry QualificationOverviewSort = ModuleID DESC,TestType ASC,TestDate ASC
  • The list will be recreated after the next test analysis.

How can one delete modules/qualifications from the list?

  • Use an sqlite editor, like sqlitebrowser or run './Controller.py -d' and then delete row by row. There will be a more user friendly way soon.

How can only new folders be analyzed, even if MoReWeb version changed?

  • Use './Controller.py -new'
